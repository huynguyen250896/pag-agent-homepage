Input Data

Pathway-level Consensus Analysis

• The first module, Pathway-level Consensus Analysis, supports three input types: (i) Expression matrix, (ii) Gene list, and (iii) Gene list with Fold change.
NOTE: Gene identifiers (IDs) in all input data types must be mapped to their corresponding unique Entrez Gene IDs before analysis. If the provided data already uses Entrez Gene IDs, this note can be skipped. Otherwise, users have two options for conversion: (i) select an ID mapping option available in PAG-Agent or (ii) upload a custom ID mapping file.



Expression Matrix

For Expression matrix, users must provide a gene expression matrix together with sample phenotype metadata, with an optional mapping file (Figure 2).


Figure 2. Example bulk transcriptomics datasets with their corresponding metadata and optional mapping files. The three examples shown (from top to bottom) correspond to the Alzheimer's disease datasets: GSE153873, GSE61196, and GSE5281. The Expression data must be organized with gene identifiers (IDs) as rows and samples as columns. The metadata must contain two required columns, sample and condition, which specify the sample IDs and their associated experimental conditions. The mapping file is optional and is used to convert probe IDs in the input data to their corresponding Entrez Gene IDs. It must contain two columns, FROM (probe IDs) and TO (Entrez Gene IDs). To ensure compatibility with PAG-Agent during data preprocessing and downstream analyses, the required column names in both the metadata and mapping files must exactly match those shown in the figure (case-sensitive).

Gene List and Fold Change

For Gene list and Fold change, users must provide at least one input file that conforms to one of the three supported formats (Figure 3a-c). The format shown in Figure 3a supports PA using ORA only, whereas the formats shown in Figures 3b and 3c enable PA using all supported statistical methods, including ORA, fast Gene Set Enrichment Analysis (fGSEA), the Kolmogorov-Smirnov (KS) test, and the Wilcoxon test.
Note that the format in Figure 3c also enables users to perform Gene-level Meta-Analysis (second module).


Figure 3. Examples of Gene list with Fold change input files accepted by PAG-Agent in two minimum formats: with or without the statistic column. The ID column contains Entrez Gene IDs, whereas p.value and statistic represent the p-values and test statistics generated by the selected DEA method. Specifically, statistic corresponds to the t-statistic for limma, the Wald statistic for DESeq2, and the likelihood ratio statistic for edgeR. Format (a) supports PA using ORA only, whereas format (b) enables all supported pathway analysis methods, including ORA, fGSEA, the KS test, and the Wilcoxon test. Format (c) is required for meta-analysis and includes the additional columns logFC (log2 fold change), logFCSE (standard error of the log2 fold change), and sampleSize (number of samples used in the DEA). To ensure compatibility with PAG-Agent, the required column names in the input files must exactly match those shown in the figure (case-sensitive).

Pathway-level Meta-Analysis

• The second module, Pathway-level Meta-Analysis, accepts lists of gene-level and pathway-level statistics (Figure 4). PAG-Agent enables users to perform meta-analysis on both gene-level and pathway-level results using one of six supported methods: Fisher's method, Stouffer's method, the minimum p-value method, the geometric mean method, addCLT, and restricted (residual) maximum likelihood (REML).

Figure 4: Lists of example gene-level (a) and pathway-level (b) statistics. Each pathway-level input file (b) must contain the following columns: ID (gene set ID), p.value (gene set p-value), normalizedScore (normalized enrichment score, NES), and sampleSize (total number of samples in the study). Each PA method computes a pathway enrichment score (ES) using its own statistical framework. For ORA, the Wilcoxon test, and the Kolmogorov–Smirnov (KS) test, the ES (equal to the NES) represents the logarithm of the ratio between the observed and expected numbers of differentially expressed (DE) genes within a pathway. A positive ES indicates that the pathway contains more DE genes than expected by chance, whereas a negative ES indicates fewer DE genes than expected. For fGSEA, the ES reflects the overall distribution of pathway genes within the ranked gene list generated from differential expression analysis. A positive ES indicates that pathway genes are preferentially enriched near the top of the ranked list, whereas a negative ES indicates enrichment near the bottom. Because the ES depends on pathway size, it cannot be directly compared across pathways. Therefore, fGSEA computes the normalized enrichment score (NES) by normalizing the ES against the mean absolute ES obtained from random permutations with the same sign. To ensure compatibility with PAG-Agent, the required column names in both the gene-level and pathway-level input files must exactly match those shown in the figure (case-sensitive).
